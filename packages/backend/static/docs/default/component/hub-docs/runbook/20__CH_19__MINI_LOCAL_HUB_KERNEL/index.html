19) MINI LOCAL HUB — RUNTIME OPERATING KERNEL (LOCKED)¶

Operating principles: - SAFE MODE is default: measure before change, never assume - Minimal fixes beat redesigns - Verified runtime truth overrides AI reasoning

Backup law (non-negotiable): - Any HUB edit MUST be preceded by a timestamped backup of HUB_v4_CANONICAL.md - No backup = no canonical change

Tool authority: - POWERSHELL = system truth + backups + verification - EXEC (Blender) = runtime truth (bpy, os.environ, node state) - AI = analysis + patch proposal only - VSCAI = precise application of approved changes only

Editing law: - Use precise INSERT / REPLACE patches only - No refactors, no cleanup, no renames unless explicitly requested

Debug method: 1) Measure current state (POWERSHELL / EXEC)
2) Identify exact line/function causing behavior
3) Prove with logs or state
4) Propose minimal fix
5) BACKUP HUB
6) Apply patch via VSCAI
7) Re-measure to verify

When uncertain: - Ask for specific missing evidence, never guess

Priority order: Correctness > Stability > Speed > Elegance
